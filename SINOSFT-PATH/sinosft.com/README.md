## 1. 身份声明 (Identity Statement)

本目录定义了 **sinosft.com** 作为 SINOSHOP 生态核心技术引擎的数字主权与技术标准。任何实现 SINOSHOP 兼容硬件、软件或固件的开发者或组织，其合规性均以此路径下的规范为准。

- **生态隶属：** 本目录是 SINOSHOP 生态系统的**核心层 (Core Layer)**，所有上层应用（如 `sftcity.com`）和愿景（如 `seas.city`）均依赖本路径定义的接口与协议。
- **资产映射：** 本目录下的编码规范 (`/specs/`) 与治理文件 (`/governance/`)，是物理模块获得 R16 认证并映射为 RWA 数字资产的 **唯一真理来源**。

---

## 2. R16-SDK 协议定义

所有宣称兼容 SINOSHOP 生态的设备、模块或系统，必须严格按照本路径定义的 **R16 编码与探针协议** 进行身份注册、状态同步与数据上链。

- **编码即身份：** 每个物理模块的唯一 ID 必须遵循本目录 `/sdk/namespace/` 中定义的命名空间规则（根命名空间：`com.sinoshop.platform.sinosft`）。
- **孪生即资产：** 模块实时运行状态（姿态、能耗、健康度）必须通过本路径定义的接口 (`/Digital_Twin/state_sync.proto`) 同步至区块链，作为 RWA 融资与结算的直接依据。
